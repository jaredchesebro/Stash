list_count parse-order: here it reports 5 because the append_list/prepend_list calls sit in the setup block above and were applied before the inline list_count ran. Place a list_count textually between an append and the deferred get and it can still report the pre-append count — list_count runs inline while append_list/get_list defer to post_parse.

cache: the cached time stays fixed on reload while the live request time at the top advances. Stash cache lives in exp_stash and is not cleared by EE's native cache flush.

build note: assertions are shown as Output-vs-Expected columns rather than inline {if} conditionals, because deferred Stash tags (get/get_list/embed) inside a conditional don't resolve at inline-eval time and can break the parser.
